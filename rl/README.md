# Reforcement Learning

Environment: refers to the visual scenes or interactive data generated by backend programs; Agent: observes the environment and makes actions based on its own strategy to maximize game scores; State: represents the agent's position in the environment, while Observation is a partial component of the state that the agent can perceive; Action: denotes the operations the agent performs within the environment; Reward: indicates the feedback the agent receives from the environment, which serves as the signal to guide the learning process and optimize the agent's policy over time.

Furthermore, Action Space: the set of actions available for the agent to choose from; Policy: a strategy function that receives the state and outputs a probability distribution over actions, typically denoted by $\pi$; Trajectory: denoted by $\tau$, a sequence of consecutive states and actions, also referred to as Episode or Rollout; Return: the cumulative sum of rewards from the current time step until the end of the game, representing the total discounted reward; Value Function: estimates the expected return starting from a given state, denoted by $V(s)$, used to evaluate the desirability of states; Q-Function: estimates the expected return starting from a given state and taking a specific action, denoted by $Q(s,a)$; Advantage Function: measures the benefit of taking a specific action compared to the average action at a given state, defined as $A(s,a) = Q(s,a) - V(s)$.

Therefore, the objective of reinforcement learning can be summarized as training a Policy neural network $\pi$ that, for all states $s$, produces actions that maximize the Return. Alternatively, it can be stated as maximizing the expected Return across all Trajectories. The core formula is $E(x)_{x \sim p(x)} = \sum_{x} x \cdot p(x) \approx \frac{1}{n} \sum_{i=1}^{n} x_{x \sim p(x)}$, which expresses the expected value of $x$ under the probability distribution $p(x)$, and can be approximated by the empirical mean over $n$ samples.

## Proximal Policy Optimization(PPO)

The core of the Proximal Policy Optimization algorithm lies in alternating between sampling data through interaction with the environment and optimizing the surrogate objective using stochastic gradient ascent, enabling multiple epochs of batch updates. PPO optimizes a novel objective with clipped probability ratios, which forms a pessimistic lower bound on the performance estimate.
